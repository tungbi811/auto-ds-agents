### 1. Competition Overview
The Titanic competition is designed as a beginner-friendly challenge to help participants learn basic machine learning concepts using a real historical event. The goal is to build a predictive model that determines which passengers survived the Titanic disaster based on various passenger features such as age, gender, and socio-economic status.

### 2. Files
Two main datasets are provided:
- **train.csv**: Contains passenger data for 891 individuals, including the target variable, "Survived" (0 for not survived, 1 for survived).
- **test.csv**: Contains data for 418 individuals without the "Survived" variable, which participants will use to make their predictions.

Additionally, a **sample_submission.csv** is included as an example format for submission.

### 3. Problem Definition
The problem is to predict the survival of passengers aboard the Titanic based on available features. Participants must create a model using the training data and then apply it to the test data to predict survival outcomes.

### 4. Data Information
#### 4.1 Data Type
- **ID Type**:
  - PassengerId: Unique identifier for each passenger.
  
- **Numerical Type**:
  - Age: Age of the passenger.
  - SibSp: Number of siblings or spouses aboard.
  - Parch: Number of parents or children aboard.
  - Fare: Fare paid by the passenger.
  
- **Categorical Type**:
  - Survived: Survival status (0 or 1).
  - Pclass: Ticket class (1 = 1st class, 2 = 2nd class, 3 = 3rd class).
  - Name: Name of the passenger.
  - Sex: Gender of the passenger (male or female).
  - Ticket: Ticket number.
  - Cabin: Cabin number.
  - Embarked: Port of embarkation (C = Cherbourg, Q = Queenstown, S = Southampton).
  
- **Datetime Type**: None.

#### 4.2 Detailed Data Description
The training dataset contains 12 features, while the test dataset has 11 features (excluding the survival outcome). Key characteristics include:
- Binary outcome for survival.
- Socio-economic indicators represented by the "Pclass" feature.
- Gender differentiation through the "Sex" feature.
- Age with potential missing values.
- Significant variation in the "Fare" feature.
- The "Embarked" feature reflecting boarding ports.

### 5. Target Variable
The target variable is **Survived**. This variable is included in the training set but not in the test set, and it is the primary output that participants will predict.

### 6. Evaluation Metrics
The evaluation metric for this competition is **accuracy**, which measures the percentage of correct predictions out of total predictions made.

### 7. Submission Format
The submission file must be in CSV format with exactly two columns:
- **PassengerId**: unique identifier for the passengers (sorted in any order).
- **Survived**: binary predictions (0 for not survived, 1 for survived).

Example format:
```
PassengerId,Survived
892,0
893,1
894,0
...
```

### 8. Other Key Aspects
- The competition is open-ended, with a rolling leaderboard that resets every two months. This encourages continuous improvement and engagement.
- Participants are encouraged to engage with the community through discussion forums and shared notebooks.
- Feature engineering can enhance model performance, and participants are encouraged to explore various modeling techniques.
